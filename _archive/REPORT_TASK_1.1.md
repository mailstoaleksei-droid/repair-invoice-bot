### 1. `file_monitor.py` (374 строки)
**Назначение:** Модуль мониторинга файловой системы

**Ключевые компоненты:**
- Класс `PDFFileHandler` - обработчик событий файловой системы
- Класс `FileMonitor` - менеджер мониторинга
- Debounce механизм (5 секунд)
- Проверка готовности файла
- Callback система для уведомлений
- Статистика обнаруженных файлов

**Технологии:**
- `watchdog` - отслеживание файловой системы
- `threading` - фоновые процессы
- `logging` - логирование событий

**Особенности:**
```python
# Автоматическое обнаружение
def on_created(self, event):
    if event.src_path.endswith('.pdf'):
        self.pending_files[file_path] = time.time()

# Debounce обработка
def _process_pending_files(self):
    for file_path, timestamp in self.pending_files.items():
        if current_time - timestamp >= 5:  # 5 сек
            self.callback(file_path, filename, file_size)
```

### Примеры

```python
# Хороший код: Type hints + Docstring
def format_file_size(size_bytes: int) -> str:
    """
    Отформатировать размер файла в человекочитаемый вид
    
    Args:
        size_bytes: Размер в байтах
        
    Returns:
        str: Отформатированный размер (например, "1.5 MB")
    """
    for unit in ['B', 'KB', 'MB', 'GB']:
        if size_bytes < 1024.0:
            return f"{size_bytes:.1f} {unit}"
        size_bytes /= 1024.0
    return f"{size_bytes:.1f} TB"
```

## 🎓 ИЗВЛЕЧЕННЫЕ УРОКИ

### Что сработало хорошо ✅

1. **Watchdog библиотека** - отличное решение для мониторинга
2. **Debounce** - критически важен для больших файлов
3. **Inline кнопки** - удобный UX в Telegram
4. **Фоновые потоки** - позволяют боту оставаться доступным

### Проблемы и решения 🔧

**Проблема 1:** Файлы обрабатывались до полной записи
- **Решение:** Debounce механизм + проверка готовности

**Проблема 2:** Бот блокировался во время мониторинга
- **Решение:** Фоновые daemon-потоки

**Проблема 3:** Duplicate обнаружения при изменении файла
- **Решение:** Set с обработанными файлами
