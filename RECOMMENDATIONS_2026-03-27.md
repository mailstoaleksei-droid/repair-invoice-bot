# Recommendations — 2026-03-27

## Главные рекомендации

1. Сначала стабилизировать текущий pipeline, потом добавлять AI.
2. Не пытаться сразу выйти на 90% автоматизации без тестов, OCR и нормальных метрик.
3. Привести локальную папку проекта к нормальному git-состоянию, иначе история изменений и совместная работа будут теряться.

## Технические рекомендации

- Вынести все пути, токены и chat id в `.env`
- Исправить запуск бота на актуальный `process_pdf_v7_3.py`
- Проверить конфигурацию на текущий год и текущий Excel-файл
- Сделать единый объект результата обработки счета с полями `supplier`, `invoice`, `date`, `truck`, `amount`, `confidence`, `source_type`
- Добавить OCR fallback для image-only PDF
- Вести structured log по причинам manual processing
- Сделать supplier registry вместо разрозненных условий по всему файлу

## Рекомендации по качеству данных

- Для каждого поставщика явно задокументировать правило:
  - какой шаблон определения поставщика
  - как искать номер счета
  - как искать дату
  - где брать сумму
  - это NETTO или BRUTTO
- Добавить regression-набор на уже известных проблемных поставщиков
- Проверять не только извлечение суммы, но и разумность суммы
- Для новых поставщиков сначала отправлять счет в review/manual, а не пытаться автоматически писать в Excel без confidence-оценки

## Рекомендации по Telegram-боту

- Бот должен проверять наличие актуального исполняемого скрипта до запуска
- Бот должен явно сообщать, какой файл и какую версию обработчика он запускает
- Ошибки конфигурации должны уходить в Telegram отдельным сообщением до старта batch
- Мониторинг файлов нужно либо вернуть в рабочую папку, либо убрать из интерфейса бота, чтобы не создавать ложное впечатление, что функция работает

## Рекомендации по GitHub

- Сделать локальный clone репозитория и работать из него
- Хранить документацию, чек-листы и changelog в репозитории
- Добавить issue-шаблоны:
  - новый поставщик
  - ошибка извлечения
  - ошибка суммы
  - проблема со сканом
- После стабилизации добавить CI хотя бы на smoke tests

## Рекомендации по этапам развития

### Этап 1

Стабилизация текущего решения:

- пути
- бот
- Excel
- метрики
- тесты на текущие правила

### Этап 2

Улучшение покрытия:

AI-расширение:
